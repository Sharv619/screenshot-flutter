# Use-Case Journey: Screenshot Action Inbox

## Overview

This document maps example journeys for Screenshot Action Inbox. The current product direction is not automatic folder sorting. The app should help users turn screenshots into structured saved items, useful actions, local search results, and optional cleanup decisions.

## Use Case 1: Place or Restaurant Discovery

**Trigger:** User manually imports or shares a screenshot of a restaurant, address, map snippet, travel post, or recommendation.

### Journey

1. **Import:** The user selects the screenshot or shares it into the app.
2. **Local Analysis:** On-device OCR extracts visible text.
3. **Entity Extraction:** Rules identify location-like text such as restaurant names, addresses, city names, map URLs, or travel keywords.
4. **Action Card:** The app shows a place/restaurant card with editable fields.
5. **User Options:**
   - Save to a places list.
   - Open the likely destination in a maps app.
   - Edit extracted fields.
   - Dismiss the suggestion.
6. **Cleanup:** After the user saves or acts, the app may ask whether to delete the original screenshot. Deletion must use native OS-level confirmation.
7. **Outcome:** The user keeps the useful place information without needing to keep the screenshot forever.

## Use Case 2: Book Research

**Trigger:** User imports or shares a screenshot of a book cover, reading list, recommendation, product page, or review.

### Journey

1. **Import:** The screenshot enters the inbox through manual selection or share-to-app.
2. **Local Analysis:** OCR extracts possible title, author, and surrounding context.
3. **Entity Extraction:** Rules identify book-like patterns.
4. **Action Card:** The app displays a book card with editable title and author fields.
5. **User Options:**
   - Save to a reading list.
   - Search externally or open a related app.
   - Enable optional enrichment to fetch metadata from Google Books API.
6. **Cleanup:** Once the book is saved, the app may offer to delete the original screenshot using OS confirmation.
7. **Outcome:** The user converts a visual reminder into a searchable reading-list item.

## Use Case 3: Receipt or Coupon Capture

**Trigger:** User imports or shares a screenshot of a receipt, order confirmation, discount code, or coupon.

### Journey

1. **Import:** The screenshot is manually selected or shared into the app.
2. **Local Analysis:** OCR extracts merchant names, dates, totals, promo codes, and expiry text where possible.
3. **Entity Extraction:** Rules classify the screenshot as a receipt or coupon candidate.
4. **Action Card:** The app shows a receipt or coupon card with structured fields.
5. **User Options:**
   - Save receipt details.
   - Save coupon code and expiry.
   - Edit any incorrect OCR fields.
6. **Cleanup:** The app may ask whether the original screenshot should be deleted after the useful information is saved.
7. **Outcome:** The user can find the receipt or coupon details later without browsing screenshots.

## Future Category Expansion

Potential future categories include events, movies, shopping products, recipes, travel plans, contacts, finance, and tickets. These should be added only after the MVP action-inbox loop works reliably for the first three category groups.
