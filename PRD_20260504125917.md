# Product Requirements Document (PRD) - Screenshot Action Inbox
**Version:** 1.1 historical planning note, revised for current product direction

## Introduction

This document previously described the product as Screenshot Auto-Categorizer. The current direction is Screenshot Action Inbox: a privacy-first app that turns screenshots into searchable, structured, actionable items.

## Updated Core Direction

- Extract useful information from screenshots locally.
- Generate action cards for likely next steps.
- Save structured details into local lists.
- Help users clean up screenshots only after explicit OS-level confirmation.
- Treat categorization as an internal engine feature, not the whole product.

## MVP Categories

- Books
- Places / Restaurants
- Receipts / Coupons

More categories can be added after the core action-inbox loop is validated.

## Cleanup Prompt

The app may suggest deleting the original screenshot after the user saves useful information or completes an action. Deletion must always use native OS confirmation:

- Android: `MediaStore.createDeleteRequest`
- iOS: `PHAssetChangeRequest`

No silent deletion is allowed.

## Privacy and Network Use

Core processing is local-only. OCR, rule-based extraction, action card generation, saved lists, and search should work without network access.

Optional enrichment mode may use external APIs only when the user enables it:

- Google Books API for book metadata.
- Google Places API for place details.

Enrichment is optional and must not be required for the core workflow.
