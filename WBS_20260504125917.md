# Work Breakdown Structure (WBS) - Screenshot Action Inbox
**Version:** 1.1 historical planning note, revised for current product direction

## Phase 1: Validation and Research

- Validate manual import and share-to-app flows.
- Research selected-photo permissions and native delete confirmation.
- Define MVP categories: Books, Places / Restaurants, Receipts / Coupons.

## Phase 2: Core App Shell

- Build inbox, review, saved lists, search, and settings screens.
- Add action card UI patterns.
- Add permission education states.

## Phase 3: OCR and Local Processing

- Integrate platform OCR.
- Normalize extracted text.
- Build deterministic entity extraction and category scoring.
- Store OCR results and structured data locally.

## Phase 4: Actions and Cleanup

- Generate book, place, receipt, and coupon action cards.
- Save useful details into local lists.
- Open relevant apps where appropriate.
- Prompt for screenshot cleanup after save/action.
- Use OS-level delete confirmation only.

## Phase 5: Optional Enrichment

- Add user-enabled enrichment mode.
- Integrate Google Books API and Google Places API only after disclosure and opt-in.
- Keep local-only mode fully functional.

## Phase 6: Optional Monitoring and Launch

- Evaluate selected-photo access and screenshot folder monitoring after MVP validation.
- Avoid background monitoring as a first milestone.
- Complete testing, privacy review, store readiness, and launch prep.
